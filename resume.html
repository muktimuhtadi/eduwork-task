GIT INIT

Untuk menginisialisasi repository atau memberi tahu folder saat akan dijadikan repository

Kita tidak bisa menggunakan git tanpa menginisialisasinya

Membuat folder

mkdir [nama folder]

menginisialisasi folder yang sudah dibuat

git init

GIT CONFIG

sebelum menggunakan git, wajib dikonfigurasi dulu

2 tipe konfigurasi: local & global

melihat semua konfigurasi di git

git config --list

konfigurasi local usernam

git config --local user.name

konfiguasi local email

git config --local user.email

kalau globaal, --local tinggal diganti --global

GIT STATUS

digunakan untuk mengecek status

membuka vs code melalui terminal git bash

code

memasukkan semua file ke dalam staging area

git add .

mengecek apakah file sudah masuk ke staging area

git status

GIT COMMIT

membuat commit

git commit -m "deskripsi"

mengece hasil commit

git log

sebelum commit harus add dulu

git add .

keluar dari log

q

membatalkan commit

git reset --hard

GIT CLONE

untuk mengkloning repo dari server (github/gitlab)

buka repo di github, copy tautan https, paste di terminal git bash dengan command:

git clone [tautan dari github]

GIT REMOTE

git remote -v

GIT BRANCH

cabang/kopian dari repo

parallel development/kolaborasi

memanajemen perkembangan

digunakan saat mengembangkan fitur baru

melihat semua branch local

git branch

melihat semua branch

git branch --all

membuat branch baru

git branch [nama-branch]

berpindah ke branch lain

git checkout [nama-branch]

menghapus branch

git branch -d [nama-branch]

menggabungkan commit dari 2 branch

git merge [nama-branch]

CONFLICT

kondisi ketika 2 programmer menggabungkan 2 branch yang memiliki perubahan pada file yang sama

ketika conflict, tidak bisa commit, harus damai dulu
